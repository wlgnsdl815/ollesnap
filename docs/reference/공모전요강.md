## ❓자주 묻는 질문 (FAQ)

#### 📑 참가 신청 및 팀 변경 관련

- **Q. 신청 내용(팀명, 서비스명) 수정이 가능한가요?**
  **A.** **팀명은 변경할 수 없습니다.** 서비스명은 1차 심사 전까지 기능설명서에 최종 변경사항을 반영하여 제출하시면 수정이 가능합니다.
- **Q. 기존에 제출했던 팀 정보(팀명, 팀원), 제안서 수정이 가능한가요?**
  **A.** **서비스 개발 기간 종료 전까지, 팀원 및 제안서 내용 수정은 가능**합니다. 이때 이미 제출한 예비심사 자료는 별도로 수정할 필요 없으며, **이후 1차 심사 자료 제출 시 접수 페이지에서 최종 변경된 팀원 정보를 다시 작성하여 반영**하시면 됩니다. 개발 기간 종료 후 1차 심사 자료 제출 시에는 이처럼 최종 변경사항이 모두 반영되어 있어야 합니다.
  ⚠ **단, 팀명과 팀대표는 수정 불가하며, 1차 심사자료 제출 이후로는 이외 항목도 일체 수정 불가**
- **Q. 신청 시 제출했던 제안서의 내용과 다른 내용으로 서비스를 개발해서 제출해도 되나요?**
  **A.** 기 제출한 제안서 내용에서 변경사항(개선, 수정 등)이 있을 경우 서비스 개발시, 반영하여 진행하시면 됩니다. 1차 심사자료 제출시, 기능설명서에 해당 부분 반영하여 제출해주시기 바랍니다.

#### 💻 데이터(API) 및 개발 방식 관련

- **Q. API 데이터를 제 서버(로컬 DB)에 저장하거나 캐싱(Caching)해서 서빙해도 되나요?**
  **A.** 한국관광공사 OpenAPI 서비스는 데이터 동기화 오류 방지를 위해 로컬 DB 저장 방식이 아닌 **실시간 호출 방식으로 활용하실 것을 강력히 권고**드립니다.
  본 공모전은 공사 OpenAPI를 필수로 활용한 서비스만을 심사 대상으로 하므로, API 활용 여부 확인을 위해 1차 심사 자료 접수 시 활용 API 운영계정 신청 정보(신청자명, 인증키 정보)를 요청할 예정입니다. 해당 정보를 기반으로 **실제 API 호출 내역을 검증**합니다.
  로컬 DB 저장 방식을 사용하여 서비스 개발 기간 내 API 호출 이력이 확인되지 않을 경우, **심사에서 불이익을 받을 수 있으니** 각 각별히 유의 부탁드립니다.
  만약 부득이한 사유로 로컬 DB 저장 방식을 활용하셔야 하는 경우에는 **별도의 신청서 작성**이 필요하며, 이 경우 아래 OpenAPI 관련 문의처(tourapi@knto.or.kr)로 추가 문의 부탁드리겠습니다.
- **Q. 한국관광공사 OpenAPI를 반드시 활용해야 하나요?**
  **A.** **네, 필수 사항입니다.** **파일 형태의 관광데이터만 활용**하여 개발하는 것은 **불가능**하며, **반드시 한국관광공사 API를 활용**하셔야 합니다. **미활용 시 심사 대상에서 제외**됩니다.
- **Q. 한국관광공사 OpenAPI 필수 활용에 대한 인정 기준은 무엇인가요?**
  **A.** 공사 OpenAPI 필수 활용 여부 판단을 위해 서비스 개발에 활용한 **한국관광공사 OpenAPI의 호출 내역, 서비스내 OpenAPI 활용 내역을 확인**할 예정입니다.
  따라서 1차 심사자료 제출시, OpenAPI 호출내역 확인을 위해 서**비스 개발에 활용한 OpenAPI의 운영계정 신청 및 신청정보를 요청할 예정**이며, 서비스 내 OpenAPI 활용내역 확인을 위해 기능설명서 내 관련 내용에 대해 작성 요청할 예정입니다.
  ⚠**OpenAPI 형태만 인정, 파일데이터 활용은 인정되지 않음**
- **Q. 개발 과정에서 생성형 AI 기술이나 AI 코딩 도구를 활용해도 되나요?**
  **A.** 제한 없이 활용하셔도 좋습니다.서비스 기능 내에 생성형 AI 기술(챗봇, RAG, 에이전트 등)을 접목하는 것은 물론, 개발 과정에서 코딩 보조 도구(Cursor, GitHub Copilot 등)를 사용하는 것에 어떠한 감점이나 제한도 없습니다.
  - 본 부문은 기술 활용 방식 자체보다, 최종적으로 안정적으로 구동되는 완제품 웹/앱 서비스를 완성해 내는 것이 평가의 핵심입니다.
- **Q. 개발 중인 서비스가 '위치기반서비스사업자' 신고 대상에 해당하나요?**
  **A.** 공모전 출품작 및 테스트 서비스라 하더라도 **법적 기준은 동일하게 적용**됩니다. 아래 가이드라인을 참고하시어 해당 여부를 **1차적으로 확인**해 주시기 바랍니다.
  🟢 **1. 신고 대상에서 제외되는 경우 (비신고)**
  - 개인위치정보를 이용자의 단말기(스마트폰, GPS단말기 등) **내부에서만 활용**하고, 사업자의 **서버나 시스템으로 전송하지 않는 경우**(예: 단말기에서 단순 조회 후, 서비스 내에서만 활용되고 즉시 삭제되는 형태)
  - ※ 개인 위치가 특정되지 않도록 데이터를 **익명 또는 가명 처리**하여 활용하는 경우, 특정 개인의 위치를 파악할 수 없는 정보이기 때문에 개인위치정보에 해당하지 않아 **신고 대상에서 제외**
  🔴 **2. 반드시 신고해야 하는 경우 (신고 대상)**
  - 서비스 제공을 위해 이용자의 개인위치정보를 **사업자의 서버로 전송**하여 활용하는 경우 **(서버 DB 저장 여부와 무관하게, 전송하는 자체만으로도 대상에 해당)**
  ⚠️ **[중요 안내]**
  - 개발 중이신 서비스의 데이터 흐름이나 시스템 구조에 따라 법적 해석이 미묘하게 달라질 수 있습니다. 주최 측에서는 개별 서비스에 대한 법적 확답을 드리기 어렵습니다.

#### 🚀 제출 및 심사(웹·앱) 관련

- **Q. 웹(Web)과 앱(App) 서비스 제출 시 유의사항이 무엇인가요?**
  **A.** \* **웹(Web) 서비스** : 별도의 승인 절차 없이 최종 서비스 URL을 제출하시면 됩니다.
  - **앱(App) 서비스 :** **제출 기한 내에 앱 스토어 등록 및 승인 완료가 필수**적입니다. 심사 기간을 고려하여 미리 준비하시기 바랍니다.
- **Q. 웹사이트, 앱 등 개발형태에 따라 심사에 영향이 있나요?**
  **A.** 아니요, 서비스 형태는 심사에 영향이 없습니다.
  참고로 웹 사이트의 경우 서비스 URL만 제출하면 되지만, **앱 서비스의 경우, 제출기한 내 앱스토어 등록 및 승인 절차 완료**가 필수이므로 해당 기간까지 고려하여 개발 완료기간을 산정해야 한다는 점 유의해주시기 바랍니다.
  ⚠ 앱스토어 등록 교육 진행 예정이므로 필요 시 참가, 앱스토어별 심사 소요기간 등 사전 확인 필수
- **Q. 지역특화서비스로 인정되는 지역 기준은 무엇인가요?**
  **A.** 지역특화서비스는 **전국 단위가 아닌 특정 지역(서울 제외)에 특화된 서비스를 의미**하며, 지역에 대한 기준은 다음과 같습니다.
  - **최대 범위:** 광역자치단체 단위 (광역시, 특별자치시, 도, 특별자치도)
  - **최소 범위:** 기초자치단체 단위 (시, 군, 구)
  📌 **참고사항:** 1개 광역자치단체 내의 여러 기초자치단체를 묶어 제안하는 것은 가능하나(예: 강원도 내 원주시·강릉시 연계), 전국 단위의 광범위한 서비스는 지역특화서비스에서 제외됩니다.
